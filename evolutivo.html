Modelo Evolutivo​

Definición: Construye el sistema de manera incremental, lanzando versiones sucesivas que van mejorando.

Mejora: El prototipo se convierte en parte del producto final.

Ventajas

  • Se adapta a cambios.
  • El cliente recibe versiones funcionales tempranas.
  • Reduce riesgos de fracaso.

Desventajas

  • Puede ser costoso por tantas iteraciones.
  • Requiere usuarios disponibles para feedback constante.

Conclusión: Ideal cuando no se conocen todos los requisitos al inicio, pero puede alargar el proyecto.
